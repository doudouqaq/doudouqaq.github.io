服务器方法

服务器

fastify.server：由Fastify的工厂函数生成的Node原生server对象。

after

当前插件及在其中注册的所有插件加载完毕后调用。总在 fastify.ready 之前执行。

ready

当所有插件的加载都完成时调用。如有错误发生，它会传递一个 error 参数。

listen

所有的插件加载完毕、ready 事件触发后，在指定的端口启动服务器。

route

将路由添加到服务器的方法，支持简写。

close

fastify.close(callback)：调用这个函数来关闭服务器实例，并触发 onClose钩子。

decorate*

向 Fastify 实例、响应或请求添加装饰器函数。

register

Fastify 允许用户通过插件扩展功能。插件可以是一组路由、装饰器或其他。

use

向 Fastify 添加中间件。

addHook

向 Fastify 添加特定的生命周期钩子函数。

prefix

添加在路由前的完整路径。

log

日志的实例。

inject

伪造 http 注入 (作为测试之用) 。

printRoutes

fastify.printRoutes()：打印路由的基数树 (radix tree)，可作调试之用。
记得在 ready 函数的内部或之后调用它。
